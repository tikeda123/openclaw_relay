# クライアント導入ガイド

## 目的

`OptionABC001` と worker ノード群が、Relay mailbox API を Bearer token 付きでそのまま使えるようにする。  
この文書は caller 側ノードに配る最小セットをまとめる。

## 配布物

- [openclaw-relay-client.env.example](../deploy/client/openclaw-relay-client.env.example)
- [relay-mailbox-send.sh](../deploy/client/relay-mailbox-send.sh)
- [relay-mailbox-worker.py](../deploy/client/relay-mailbox-worker.py)

## caller 側で置くもの

前提:

- Relay 側の `relay.health_host` は `0.0.0.0` または caller から到達できる LAN IP にする
- token 認証なしで mailbox API を LAN 公開しない

### 1. 環境変数ファイル

任意の場所に env file を置く。例:

```bash
mkdir -p ~/.config/openclaw-relay
cp deploy/client/openclaw-relay-client.env.example ~/.config/openclaw-relay/client.env
```

設定する値:

```bash
RELAY_BASE_URL=http://relay-host:18080
RELAY_MAILBOX_TOKEN=<caller mailbox token>
```

`OptionABC001`、`OptionDEF002`、`OptionGHI003` では `RELAY_MAILBOX_TOKEN` の値だけが違う。

### 2. script

caller 側に script を置いて実行権限を付ける。

```bash
mkdir -p ~/.local/bin
cp deploy/client/relay-mailbox-send.sh ~/.local/bin/
cp deploy/client/relay-mailbox-worker.py ~/.local/bin/
cp deploy/client/relay-mailbox-injector.py ~/.local/bin/
chmod +x ~/.local/bin/relay-mailbox-send.sh ~/.local/bin/relay-mailbox-worker.py ~/.local/bin/relay-mailbox-injector.py
```

script は既定で `~/.config/openclaw-relay/client.env` を自動読込する。  
別の env file を使うときだけ `RELAY_CLIENT_ENV=/path/to/client.env` を付ける。

## 使い方

### OptionABC001 から送る

```bash
relay-mailbox-send.sh \
  --to OptionDEF002 \
  --body "現在の稼働状況を1行で返してください。"
```

### worker が受ける

手動受信はしない。worker では mailbox worker 常駐 process が `GET /v1/messages` を破壊的取得し、そのまま local OpenClaw に投入する。

### worker が返信する

返信も新しい message として送る。

```bash
relay-mailbox-send.sh \
  --to OptionABC001 \
  --body "稼働中です。応答正常です。" \
  --conversation-id CONV-001 \
  --in-reply-to MSG-001
```

## 運用ルール

- caller は `Authorization: Bearer <mailbox token>` を自分で付ける
- `from` は caller が書かない
- `from` は Relay が token から補完する
- `GET /v1/messages` は破壊的取得なので、受信したら即処理する
- 手動 `GET /v1/messages` や `relay-mailbox-receive.sh` は通常運用で使わない
- 「届いたか」の確認は Human Chat だけでなく、inject 済み session、worker / injector log、Relay audit を見る
- mailbox API は pull-only なので、worker host と OptionABC001 host の両方で常駐 process を動かす

## worker 常駐化

### 単発実行

```bash
relay-mailbox-worker.py --once --verbose
```

動作:

- `GET /v1/messages`
- local OpenClaw `/v1/responses`
- `PUT /v1/messages` で reply
- 既定では human-facing `agent:main:main` session に入れる

### macOS launchd

`OptionDEF002` や `OptionGHI003` のような macOS worker では LaunchAgent を使う。

```bash
mkdir -p ~/Library/LaunchAgents ~/Library/Logs
sed "s#__HOME__#$HOME#g" \
  deploy/client/com.openclaw.relay-mailbox-worker.plist.template \
  > ~/Library/LaunchAgents/com.openclaw.relay-mailbox-worker.plist
launchctl bootout "gui/$(id -u)" ~/Library/LaunchAgents/com.openclaw.relay-mailbox-worker.plist 2>/dev/null || true
launchctl bootstrap "gui/$(id -u)" ~/Library/LaunchAgents/com.openclaw.relay-mailbox-worker.plist
launchctl kickstart -k "gui/$(id -u)/com.openclaw.relay-mailbox-worker"
```

確認:

```bash
launchctl print "gui/$(id -u)/com.openclaw.relay-mailbox-worker" | head -n 40
tail -f ~/Library/Logs/openclaw-relay-mailbox-worker.log
```

## OptionABC001 受信常駐化

`OptionABC001` は Relay inbox の reply を自動で取り込み、OpenClaw の human-facing `agent:main:main` session に inject する。

### 単発実行

```bash
relay-mailbox-injector.py --once --verbose
```

動作:

- `GET /v1/messages`
- local OpenClaw `/v1/responses`
- 既定では human-facing `agent:main:main` session に inject する
- inject 成功後、`notifyHuman=true` の reply で Telegram delivery context があれば固定フォーマットで direct notify する
- inject 失敗時は `~/.local/state/openclaw-relay/mailbox-injector-deadletter.jsonl` に保存

### macOS launchd

`OptionABC001` のような macOS orchestrator では LaunchAgent を使う。

```bash
mkdir -p ~/Library/LaunchAgents ~/Library/Logs
sed "s#__HOME__#$HOME#g" \
  deploy/client/com.openclaw.relay-mailbox-injector.plist.template \
  > ~/Library/LaunchAgents/com.openclaw.relay-mailbox-injector.plist
launchctl bootout "gui/$(id -u)" ~/Library/LaunchAgents/com.openclaw.relay-mailbox-injector.plist 2>/dev/null || true
launchctl bootstrap "gui/$(id -u)" ~/Library/LaunchAgents/com.openclaw.relay-mailbox-injector.plist
launchctl kickstart -k "gui/$(id -u)/com.openclaw.relay-mailbox-injector"
```

確認:

```bash
launchctl print "gui/$(id -u)/com.openclaw.relay-mailbox-injector" | head -n 40
tail -f ~/Library/Logs/openclaw-relay-mailbox-injector.log
```

log には次が出る。

- `processed mailbox reply=...`
- `sent telegram notice reply=... chat=... telegram_message_id=...`

## 確認

### 送信確認

```bash
relay-mailbox-send.sh --to OptionDEF002 --body "smoke"
```

`messageId` を含む JSON が返れば送信受理。

新しい worker 追加の具体例は [新規worker追加手順.md](新規worker追加手順.md) を参照。

人間への非同期報告も試す場合:

```bash
relay-mailbox-send.sh --to OptionDEF002 --notify-human --body "smoke"
```

### 受信確認

- `OptionDEF002` は `~/Library/Logs/openclaw-relay-mailbox-worker.log` を見る
- `OptionABC001` は `~/Library/Logs/openclaw-relay-mailbox-injector.log` と human-facing `agent:main:main` session を見る
- Telegram 側の固定通知は `notifyHuman=true` のときだけ injector が direct send するので、AI の会話応答を待たない
- Relay host では `api/dashboard`、audit、DB の `messageId` を見る
