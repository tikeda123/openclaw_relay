# 運用監視画面

## 目的

会話画面とは分離して、Relay の運用状態だけを見る。

- active alerts
- worker health
- retry / deadletter
- dispatch / inject latency
- deadletter message 一覧

## アクセス

- `http://127.0.0.1:18080/ops`
- `http://127.0.0.1:18080/monitor`

JSON は会話画面と同じ:

- `http://127.0.0.1:18080/api/dashboard`

## 画面構成

### Overview

- ready
- active alerts
- done
- deadletter
- source sync
- node / alert webhook 最終受信時刻

### Workers

worker ごとに次を表示する。

- tunnel / http
- done / in-flight
- retry-a / retry-b
- dead-a / dead-b
- dispatch avg / latest
- inject avg / latest

### Active Alerts

Alertmanager から Relay に届いている active alert を表示する。

### Status Matrix

worker ごとの `DONE / RESERVED / FAILED / DEADLETTER` 件数を表で見る。

### Deadletters

直近の deadletter message を worker, task, status, updated, error 付きで表示する。

## 補足

- `/ops` は会話タイムラインを出さない
- 会話確認は `/` または `/ui`
- 5 秒ごとに自動更新する
