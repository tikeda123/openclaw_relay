# 運用ダッシュボード

## 目的

OptionABC001 と OptionDEF002 の往復を、時系列チャットとして読む。UI は会話を優先し、補助情報は上部の最小ステータスだけに絞る。

Relay DB にある request / reply だけでなく、`OptionABC001` の recent session log から `OptionDEF002` 関連メッセージも取り込み、実際に出しているタスク内容が見えるようにする。

## アクセス

Relay の health server と同じポートで UI を出す。

- `http://127.0.0.1:18080/`
- `http://127.0.0.1:18080/ui`
- 監視専用画面: `http://127.0.0.1:18080/ops`

JSON API:

- `http://127.0.0.1:18080/api/dashboard`

## 画面構成

### Conversation Feed

- request / reply を時系列の chat bubble で表示
- 左が OptionABC001、右が OptionDEF002
- pending / error は Relay bubble として会話の途中に差し込む

### Summary Strip

- ready / latest poll / done / in-flight / deadletter / watch pending

### 別画面との分離

- `/` と `/ui`
  - 会話モニタ専用
- `/ops`
  - 運用監視専用
  - worker health / retry / deadletter / latency / active alerts を見る

### 画面から外したもの

- Recent Audit
- Live Log
- 右側サイドバー

## 補足

- UI は read-only。
- `health_host = 127.0.0.1` のままなら、Relay サーバ上で見るか、SSH port-forward で見る。

例:

```bash
ssh -L 18080:127.0.0.1:18080 relayuser@relay-node
```

その後、手元ブラウザで `http://127.0.0.1:18080/` を開く。
